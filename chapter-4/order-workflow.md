4.2 管理者的標準出貨流程

4.2 Standard Admin Order-Processing Workflow

這是在您的電商教學中，最重要的自動化流程之一。
當客戶使用「信用卡」或「LINE Pay」付款，並選擇「RY Tools 超商取貨」時，您的標準處理流程如下：

步驟 1：新訂單通知 (LINE Notify)

動作： 顧客完成付款的瞬間，您的手機 LINE Notify 會收到「新訂單通知」。

您需要做的： 無。這是自動的。

步驟 2：登入後台，確認訂單詳情

動作： 登入 WordPress 後台，前往「WooCommerce」>「訂單」。

您需要做的：

找到該筆新訂單，它的狀態應該是「處理中」（代表已付款）。

點進訂單詳情，檢查顧客購買的商品、數量是否正確，以及是否有特殊備註。

步驟 3：出貨處理 (列印寄件單)

動作： 這是您主要的手動工作。

您需要做的：

在「訂單詳情」頁面的右上方（或右側邊欄），找到**物流外掛（例如：RY Tools）**的控制盒。

點選「取得超商寄件單」或「列印託運單」的按鈕。

系統會產生一張包含寄件條碼的 A4 頁面（例如 7-11 或全家的寄件單）。

將這張單子用印表機列印出來。

步驟 4：寄件 (到超商)

動作： 將商品包裝好，把剛才列印的寄件單貼在包裹上。

您需要做的：

將包裹拿到您設定的超商（7-11 或全家）。

將包裹交給店員刷條碼。

完成。

步驟 5：系統自動化流程 (您不需動作)

這就是串接的價值所在。當您完成步驟 4 之後，所有系統會自動接手：

物流追蹤：

超商店員刷條碼後，RY Tools 會自動偵測到包裹狀態變為「運送中」。

顧客收到超商取貨簡訊並「完成取貨」後，RY Tools 會再次偵測，並自動將您後台的訂單狀態從「處理中」更新為「已完成」。

自動開立發票：

綠界科技 (ECPay) 的發票外掛會一直偵測訂單狀態。

當它發現訂單狀態變為「已完成」時，就會自動開立電子發票，並 E-mail 給顧客。

總結：
身為管理者，您唯一需要做的手動工作就是「登入後台 -> 列印寄件單 -> 拿去超商寄」。
剩下的對帳、通知、開發票，全部由系統自動完成。
