# 第 6 章：(重點) 訂單管理與出貨流程 (建議 2.5 小時)

在第 5 章，我們已經成功讓顧客可以「自動付款」並「自動開立發票」。本章節的目標，是讓您**精通「訂單成立後」的所有後台營運工作**。

這不只是「出貨」而已，還包含了如何看懂訂單狀態、如何處理客訴退款、如何匯出訂單報表給會計... 這是您身為經營者每天都必須接觸的指揮中心。

#### 在本章節，您將會學到：

1. **訂單狀態全解析：** 深入理解「保留」、「處理中」、「已完成」分別代表什麼意義？它們會觸發什麼自動化流程？
2. **(實作) 標準出貨流程 (SOP)：** 從「收到 LINE 通知」到「列印寄件單」，再到「超商寄件」的完整 SOP。
3. **(實作) 退款與退貨：** 如何在後台執行「退款」？「部分退款」怎麼做？以及「已開立的發票」該如何作廢？
4. **(實務) 訂單匯出：** 如何將訂單資料匯出成 .CSV 檔案，交給會計師報稅，或用於行銷分析。
5. **(實務) 顧客溝通：** 如何使用「訂單備註」主動通知顧客，以及如何整合「LINE Chatz」即時回覆顧客問題。

完成本章後，您將有能力流暢且專業地處理所有日常訂單，並有效解決客訴問題。
