# 5.7 (實務) 整合 Order Notify (管理者新訂單通知)

但在 2025 年的台灣，**Email 太慢了**。當顧客下單時，您（經營者）需要的是「**立即、即時**」的通知，才能在第一時間掌握訂單。

本章節的目標，就是實作台灣電商經營者**必備**的功能：**串接「Order Notify for WooCommerce」外掛，讓新訂單在成立的瞬間，立刻推播到您個人的 LINE Notify！**

我們將會使用您（Moksa）提供的專業 Flex Message 範本，打造一個資訊豐富的通知卡片。

#### 課程前提：

* 您必須有一個個人的 LINE 帳號。
* 本教學基於「**Order Notify for WooCommerce**」外掛（由 Oberon Lai 開發，舊稱 LINE Notify）。

#### 步驟 1：取得您的「LINE Notify 權杖 (Token)」

在串接外掛之前，您必須先從 LINE 官方取得一組「**權杖 (Token)**」，這就像是您和 LINE 機器人之間的「通關密語」。

1. 開啟瀏覽器，前往 LINE Notify 官方網站：`https://notify-bot.line.me/zh_TW/`
2. 點選右上角「**登入**」，使用您個人的 LINE 帳號密碼登入。
3. 登入後，點選右上角的「**個人頁面**」。
4. \[示意圖：LINE Notify 個人頁面]
5. 往下滑，找到「**發行權杖 (Generate token)**」區塊，點選「**發行**」按鈕。
6. **權杖名稱：** 填入一個您能辨識的名稱，例如：「**Moksa 網站訂單通知**」。
7. **選擇接收通知的聊天室：** (關鍵)
   * **強烈建議**選擇「**透過1對1聊天接收LINE Notify的通知**」。
   * 這樣新訂單通知才會「**只傳給您自己**」。
   * （如果您選群組，整個群組都會看到訂單通知，較不建議）。
8. \[示意圖：選擇「1對1聊天接收」]
9. 點選綠色的「**發行**」按鈕。
10. 系統會產生一組\*\*「權杖」\*\*（例如：`aBcDeFgHiJkLmN12345...`）。
11. **(極度重要)** **請立刻點選「複製」**，並將這組權杖**貼到您的記事本或密碼管理工具上**。
12. **這個權杖只會顯示這一次！** 關閉這個視窗後，您就再也無法找回它，只能重新發行。

#### 步驟 2：安裝並設定 Order Notify 外掛

1. 回到您的 WordPress 後台。
2. 前往「外掛」>「安裝外掛」，搜尋「**Order Notify for WooCommerce**」。
3. 安裝並「**啟用**」它。
4. 啟用後，前往「**WooCommerce**」>「**Order Notify**」設定頁面。

#### 步驟 3：設定「管理者通知」

1. 在 Order Notify 的設定頁面，切換到「**管理者通知**」(Admin Notification) 頁籤。
2. \[示意圖：Order Notify 的「管理者通知」頁籤]
3. **啟用管理者通知：** **(必勾選)**
4. **LINE Notify 權杖：**
   * **(必填)** 將您在**步驟 1** 複製的那一長串「權杖」，**貼到這個欄位**。
   * （您可以貼上多組權杖，用「逗號」隔開，就能同時通知多人，例如：您和您的倉管人員）。
5. **發送時機：**
   * (建議勾選) 「**訂單狀態為「處理中」**」 (顧客已付款)。
   * (建議勾選) 「**訂單狀態為「保留」**」 (顧客下單但未付款，例如 ATM)。
   * 這樣無論顧客是否付款，您都能在第一時間掌握。
6. **通知訊息範本：**
   * 預設是「簡易文字」，但我們要用更專業的「**Flex Message**」。
